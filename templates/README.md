# Exchange Server Planning Templates (F23)

Pre-built Word templates for Exchange Server planning and approval projects.

## Files

| File | Language | Purpose |
|---|---|---|
| `Exchange-concept-template-DE.docx` | German | Concept and Approval Document (German edition) |
| `Exchange-concept-template-EN.docx` | English | Concept and Approval Document (English edition) |

## Usage

1. Copy the template for your project language.
2. Rename to e.g. `Contoso_Exchange-concept_v1.0.docx`.
3. Fill in all **content controls** (blue placeholder text) using Word or LibreOffice.
4. Complete the **Questionnaire (Chapter 15)** — all open items from Chapters 3–14.
5. Collect signatures on the **Approval Page (Chapter 16)** before proceeding.
6. Attach the signed document to your change record / project folder.

## Notes

- **Exchange SE only**: Exchange 2016 and 2019 reached End-of-Support on 14 October 2025 and are no longer recommended for new deployments.
- **Chapter 10** (Migration) is conditional — remove or mark "N/A" if no legacy Exchange environment exists.
- The `[LOGO]` placeholder in the page header should be replaced with your company logo.
- Classification is set to `INTERN` — adjust in the footer if your organisation uses different classification levels.

## Regenerating the Templates

The templates are generated by `tools/Build-ConceptTemplate.ps1` using a pure PowerShell OpenXML engine.
Run as the maintainer after structural changes to the template, then commit the updated `.docx` files:

```powershell
.\tools\Build-ConceptTemplate.ps1
git add templates/
git commit -m "templates: regenerate concept templates DE + EN"
```
